# cucumber-dagger-processor

The annotation processor module for dagger-cucumber.

## What this module generates

During compilation the processor scans your test sources for `@CucumberDaggerConfiguration` and produces the following classes:

| Generated class                    | Purpose                                                                                                                                                                                   |
|------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `GeneratedScopedModule`            | A Dagger `@Module` that includes all user-provided modules containing `@ScenarioScope` bindings.                                                                                          |
| `GeneratedScopedComponent`         | A `@ScenarioScope @Subcomponent` that exposes a provision method for every scenario-scoped type and every step definition class.                                                          |
| `CucumberDaggerModule`             | A Dagger `@Module` that declares `GeneratedScopedComponent` as a subcomponent and bridges its builder to the `ScenarioScopedComponent.Builder` interface expected by the runtime.         |
| `GeneratedCucumber{YourComponent}` | A wrapper `@Component` that combines your application modules with `CucumberDaggerModule` and implements `CucumberDaggerComponent`. This is the component Dagger instantiates at runtime. |
| `GeneratedComponentResolver`       | A type-dispatching implementation of `ComponentResolver` that replaces all runtime reflection with `if (type == X.class)` chains emitted at compile time.                                 |

A `META-INF/services` entry is also written so that the runtime locates the generated component via the standard Java `ServiceLoader` mechanism.

## Adding the dependency

```kotlin
testAnnotationProcessor("dev.joss:cucumber-dagger-processor")
```

## dagger-compiler requirement

`dagger-compiler` must also be on the annotation processor classpath so that Dagger can process the interfaces and modules generated by this processor. Without it, the Dagger `@Component` implementations will not be generated and the build will fail.

```kotlin
testAnnotationProcessor("com.google.dagger:dagger-compiler:<dagger-version>")
```
